What is DOM, Shadow DOM and Virtual DOM

DOM

We can access them via the DOM API like this one document.body.p

Shadow DOM

Here we have custom elements like in the video tag such as the play button that is not accessible through the dom API

If you inspect the buttons of the video, you will see they are not available as an object of the DOM

This video component is a custom component created by the browser through the Shadow DOM

Chrome, for instance, does not show the shadow DOM by default, but you can enable them through its settings

Virtual DOM

DOM Manipulation is heavy

Sometimes, changing variables them applying them immediatly on the DOM can be very slow, specially when nowadays modern frontend websites are JS heavy.

So, to counter that, some frameworks create a virtual DOM, which is an abstraction of the DOM, where an ideal representation of the DOM is kept in memory than synced with the actual DOM

React.JS is one of the frameworks that have a Virtual DOM to boost performance and cross-browser compatibility
